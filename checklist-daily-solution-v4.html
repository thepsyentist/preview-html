체크리스트 반복 점검

현재 문제 4가지를 정확히 파악하고, 설정 화면부터 TBM 반영까지 전체 흐름을 완벽하게 설계합니다.

Part 1
지금 뭐가 문제인가
현장에서 실제로 겪는 불편함 4가지를 정확히 짚습니다.
문제 1

체크리스트를 찾아가기 어렵다

체크리스트를 보려면 사이드바 → 위험성평가 회의록 → 목록에서 해당 행 찾기 → "체크리스트 보기" 클릭까지 4단계를 거쳐야 합니다.

사이드바 → 회의록 메뉴 → 목록에서 찾기 → 체크리스트 보기
사이드바에 체크리스트 직접 메뉴가 없음
문제 2

TBM 작성 시 항목이 너무 많다

자동선택이 모든 업체의 중점관리 항목을 보여줍니다. 10개 업체 현장이면 수십 개 항목이 한번에 나와서, 내 업체 것만 고르기 힘듭니다.

현재 자동선택 필터:
{ keyIssues: true, excludeCompleted: true }
← 업체(company) 필터가 없음!
문제 3

매일 같은 점검인데 매번 새로 해야 한다

어제 TBM에서 선택한 체크리스트를 오늘 다시 쓰려면, 처음부터 다시 선택해야 합니다. "이전 복사"나 "반복" 기능이 없습니다.

1/6(월) TBM → 1/7(화) TBM → 1/8(수) TBM
매번 처음부터!
문제 4

이번 주에 뭘 점검했는지 모른다

체크리스트 상세 페이지에서는 조치 상태만 보입니다. "이번 주에 매일 이 항목을 점검했는가?"를 확인할 방법이 없습니다. 1:1 구조라 일자별 이력이 없습니다.

각 설정이 하는 일

반복 점검 ON/OFF
OFF면 현재와 동일하게 동작합니다. ON으로 켜야 아래 설정이 활성화됩니다.
점검 기간
이 기간 동안만 자동 연결됩니다. 기간이 지나면 자동으로 비활성화됩니다. 기본값: 위험성평가 관리기간과 동일.
자동 연결 대상
TBM 일지, 일일점검, 주간점검 중 어디에서 이 체크리스트를 사용할지 선택합니다. 필요 없는 곳은 체크 해제하면 됩니다.
업체 자동 필터
ON이면 TBM 작성자의 소속 업체 항목만 보여줍니다. 10개 업체 현장에서도 내 업체 3~5개 항목만 자동 선택됩니다.

TBM 저장할 때 일어나는 일

TBM을 저장하면, 어떤 항목을 점검했는지가 체크리스트 상세의 "점검 일정" 탭에 자동으로 기록됩니다.
→ 나중에 체크리스트 상세에서 "1월 8일에 TBM에서 3개 항목 중 3개 완료" 같은 이력을 볼 수 있습니다.
→ 이것이 STEP 3: 일자별 진행 이력입니다.
판단 기준
다른 도메인에도 필요한가?
점검 일정 설정의 "자동 연결 대상" 체크박스에서 선택하면 됩니다. 강제가 아닙니다.
도메인 작성 빈도 왜 필요한가 / 왜 불필요한가 필요도 기본 설정
TBM 일지 매일 필수. 매일 작성하는데 매번 체크리스트를 새로 선택해야 함. 반복 설정이 가장 절실한 곳. ★★★ 기본 체크
일일점검 매일 유용함. 매일 작성하므로 TBM과 같은 이유로 유용. participatingCompanies(참여업체)로 더 정확한 필터 가능. ★★☆ 기본 체크
주간점검 주 1회 선택적. 주 1회라 반복 필요성 낮음. 대신 해당 주의 TBM/일일점검 진행 이력을 합산해서 보여주면 더 유용. ★☆☆ 기본 해제
결론: 다른 도메인에서 isDaily가 "필요한지" 고민할 필요가 없습니다.
점검 일정 설정의 "자동 연결 대상"에서 체크/해제로 선택하면 됩니다.
TBM은 기본 체크, 주간점검은 기본 해제. 현장 상황에 따라 관리자가 조절합니다.

문제 해결 어디서
체크리스트 찾아가기 어려움 TBM에 바로가기 링크 + 사이드바 "활성 체크리스트" 메뉴 추가 사이드바, TBM

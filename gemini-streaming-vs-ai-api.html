0개요 — 두 갈래

safe-u works 에는 성격이 완전히 다른 두 개의 AI 경로가 공존한다. 핵심 차이는 어디로 가느냐(백엔드 경로) + 응답을 어떻게 받느냐(프로토콜).

① 일반 AI API

표준 ApiDefinition(axios)로 m6.api 의 /ai/safeu/... 를 호출 → 외부 AI 서버 프록시. 완성된 결과를 한 방에 받는다.

packages/api → m6.api /ai/safeu → ai.mindlens.kr (OpenAI)

② Gemini Streaming

raw fetch + SSE 로 별도 Agent 서버 /v3/safeu/agent/.../stream 호출 → Gemini 직접. 토큰을 청크 단위로 흘려받는다.

raw fetch → m6.api agent/safeu → Gemini SDK

1일반 AI API (non-stream)

다른 CRUD API 와 동일한 패턴. request → 완성 응답 (request-response).

항목내용
호출 방식표준 ApiDefinition (axios) — CRUD API 와 동일
백엔드AiSafeUService → axios.post 외부 AI 서버 프록시
모델gpt-4o 기본 (예: generateRiskAssessmentApiDef)
응답한 방에 전체 JSON 리턴
용도위험성평가 생성, 사진 분석, 체크리스트 생성, 회의록 요약 — 구조화된 큰 결과물
클라이언트useQuery/mutation 패턴, 로딩 스피너

예시 ApiDef

// packages/api/src/riskAssessment/generateRiskAssessmentApiDef.ts
export const generateRiskAssessmentApiDef: ApiDefinition<...> = {
  method: 'post',
  url: () => '/ai/safeu/works/risk-assessment/generate',
  config: ({ task, ... }) => ({
    data: { task, ..., model: model || 'gpt-4o' },
  }),
};

대상: risk-assessment/generate, photoReport/analyzePhotos, meetingNote/summarize, checklist/generate 등.

2Gemini Streaming (SSE)

클라가 fetch 의 ReadableStream 을 직접 소비하며 토큰이 도착하는 대로 화면에 흘린다.

클라 raw fetch→ m6.api /v3/safeu/agent/.../stream→ Gemini generateContentStream()
항목내용
호출 방식ApiDefinition 안 씀. raw fetch + ReadableStream 직접 소비 (async function*)
백엔드@google/generative-ai SDK → model.generateContentStream()
모델gemini-2.5-flash (WRITER_MODEL), thinkingBudget: 0
응답SSE 청크 스트림 — event: chunk / done / error
용도AI 작성 보조(조치내용 한 문장) — 온열점검, 회의록, 위험체크리스트
클라이언트커스텀 hook + 타자기 효과 UI

SSE 이벤트 포맷

event: chunk  data: { text: <delta> }   ← 토큰 단위 도착
event: done   data: { full: <final> }
event: error  data: { message }

백엔드 (NestJS 컨트롤러)

4왜 나눴나

📦 큰 구조화 결과물(위험성평가 표, 사진분석) → 부분 표시 의미 없음 → non-stream

✍️ 짧은 자연어 작성(조치내용 1문장) → 즉각 반응성 중요 → streaming + 타자기 효과

🏗️ Agent 경로는 별도 서버(포트 6600 / NEXT_PUBLIC_AGENT_API)로 분리 — m6.api 의 일반 AI 프록시와 인프라 자체가 다르다.

5타자기 효과 트릭

문제 — Gemini 가 큰 청크 단위로 텍스트를 보내서 그대로 textarea 에 붙이면 "복붙"처럼 보인다.

해결: 클라가 서버 청크를 target 버퍼에 누적하고, 별도 ticker 가 한 글자씩 displayed 로 옮겨 textarea 에 흘려보낸다.

// useHeatCheckActionAiWriter.ts (L123-175)
let target = '';      // 서버 청크 누적
let displayed = '';   // 화면에 흘린 글자
const TYPE_INTERVAL_MS = 22;   // 평균 글자 간격
const PUNCT_PAUSE_MS = 70;     // 문장부호 직후 짧은 휴지

const tick = () => {
  if (displayed.length < target.length) {
    const nextChar = target.charAt(displayed.length);
    displayed = target.slice(0, displayed.length + 1);
    onChange(displayed);
    const delay = /[\.,。·!?]/.test(nextChar) ? PUNCT_PAUSE_MS : TYPE_INTERVAL_MS;
    typewriter = setTimeout(tick, delay);
  } else if (serverDone) {
    finishWith(target);   // 다 흘리고 종료
  } else {
    typewriter = setTimeout(tick, WAIT_FOR_CHUNK_MS); // 다음 청크 대기
  }
};

Part 1
왜 정리가 필요한가
현재 SafeU Works DB에 존재하는 4가지 구조적 문제를 정의합니다.
문제 1

중복 현장 난립

협력사가 원청 현장에 합류하지 않고 자체적으로 같은 현장을 새로 생성하고 있습니다. 같은 공사현장인데 회사마다 별도로 만들어서 데이터가 파편화됩니다.

예시: "쿠팡 제천 물류센터" → 13개 중복 현장 존재
같은 현장인데 회사마다 별도 생성 → 데이터 파편화
문제 2

Corporation 구조 부재

v2 이전에 만들어진 현장은 현장소장/안전관리자 권한만 존재합니다. 하나의 회사(예: 보미건설)가 12개 현장을 운영하지만 통합 관리가 불가능합니다.

현상: 각 현장마다 company가 별도로 생성
본사 차원의 통합 관리 불가
문제 3

회사-현장 관계 미정리

하나의 고객사가 여러 현장을 갖고 있지만 시스템적으로 연결되지 않습니다. 대표관리자가 모든 현장을 한눈에 볼 수 없는 구조입니다.

문제: 보미건설 12개 현장이 각각 독립 운영
대표관리자 대시보드 구성 불가
문제 4

데이터 정합성 문제

협력사 참여자들이 중복 현장에 분산 등록됩니다. 원본 현장에는 참여자가 많지만, 중복 현장에는 1명만 있는 경우가 대부분입니다.

결과: 통합 관리/통계가 불가능
현장별 인원 현황 파악 불가

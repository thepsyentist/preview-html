결정 2 syntax highlight 위치 — 메인 vs worker
버려진 선택
메인 스레드에서 실시간 highlight
→
고른 선택
worker pool 로 지연 처리
저자 논리: highlight 는 CPU heavy. 메인 막으면 스크롤 끊김. worker 로 빼면 *직렬화 비용* 새로 생기지만 인터랙션 부드러움 우선.
스스로 답

현재 본인 프로젝트에서 *메인 스레드 점유율 높은* 작업 1개 고르면? 그게 worker 로 빠질 수 있음? 못 빠진다면 왜?

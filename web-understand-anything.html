코드를 탐색·검색·질문 가능한 인터랙티브 지식 그래프로 변환. Tree-sitter (구조) + LLM (의미) 하이브리드 파이프라인.

✨ 핵심 기능 8가지

  • 🕸️ 인터랙티브 지식 그래프 — 노드 기반 탐색. 파일·함수·클래스 단위로 시각화.
  • 🏢 비즈니스 로직 매핑 — 도메인·플로우·프로세스 단계 가시화.
  • 📚 지식베이스 분석 — Karpathy 패턴 LLM Wiki 지원.
  • 🧭 아키텍처 가이드 투어 — 의존성 순서로 자동 정렬된 안내.
  • 🔎 퍼지·시맨틱 검색 — 코드 전반에 걸친 의미 검색.
  • 🔀 Diff Impact 분석 — 변경 사항의 파급 효과 미리보기.
  • 📐 레이어 시각화 — 아키텍처 그룹 자동 인식.
  • 🎭 페르소나 적응형 UI — 사용자 유형에 따라 복잡도 조절.

⌨️ 주요 명령어

/understand 전체 코드베이스 분석
/understand-dashboard 인터랙티브 시각화 열기
/understand-chat <query> 코드에 대해 질문
/understand-diff 현재 변경분 영향 분석
/understand-domain 비즈니스 도메인 지식 추출
/understand-knowledge <path> Wiki 지식베이스 분석

👥 팀 협업

생성된 그래프는 저장소에 .understand-anything/ 디렉토리로 커밋 가능. 팀원은 비싼 파이프라인 단계를 건너뛰고 바로 그래프 활용.

💡 팁

그래프 크기가 10MB 초과하면 Git LFS 사용 권장.

🎯 적용 인사이트

  • ① 새 코드베이스 온보딩 — 신규 팀원이 며칠 걸려 잡던 전체 구조를 그래프로 단축.
  • ② 변경 영향 분석 — /understand-diff 로 버그 fix·리팩토링 시 파급 효과 미리 확인.
  • ③ LLM Wiki 지식 그래프화 — /understand-knowledge 로 문서 기반 학습 자료를 탐색 가능한 구조로.
  • ④ 비즈니스 로직 가시화 — /understand-domain 으로 도메인 흐름을 코드에서 추출. PM·기획자와 협업에 유용.
  • ⑤ 토큰 비용은 점검 필수 — 첫 실행 비용 큼. 소규모 프로젝트로 사전 검증 후 큰 코드베이스 적용.
  • ⑥ 아키텍처 약점 보강 — 전체 구조 파악이 약한 개발자에게 가장 효과적. 가이드 투어가 의존성 순서로 자동 정렬.
